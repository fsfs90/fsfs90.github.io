有了 ai 编程以后，我解决我业余项目的 bug 基本上就是 2、3 轮对话的事儿。没有了之前为了 debug，超长时间的查资料、重复试，一下几个小时持续。

docker-compose 原来也需要 build，up down 不会重新 build，所以修改代码的修改不会生效。这个 bug 几轮对话改好了。把我的 simple rss reader 项目放到 docker 里运行，只在 codex 上输入“生成 docker-compose.yml，试这个项目在 docker 中运行，和我讨论细节实现”。然后确认了几个问题，生成了 docker-compose 文件，运行时的错误复制粘贴到 chatgpt，精准定位到了问题，是 npm run build 的错误。还有缺少 typescript 的依赖。之后 --dev install 之后，又出错，又贴到 chatgpt，又精准地定位问题为需从 dev dependency 变成 dependency。这放到过去，肯定是比现在要更大费周章。

这种精准的定位问题，我猜并不是 ai 有过人的聪明，而是ai比搜索引擎能更好地理解关键词、然后匹配已有的技术文档，这种提升是肉眼可见的、确实是巨大的进步。比如，我感到现在chatgpt的功能就像一个超级聪明的stackoverflow searchbar，能从成千上万的页面中找到最符合我描述的Q and A。

我不在编程工作一线了，不知道公司里是如何的，我感到这种效率的提升必然伴随程序员总数需求的下降，想到最近的裁员，然后想到下围棋的棋手的沮丧，没想到ai饶过谁，现在很快就轮到程序员的沮丧了，我已经很久没学编程的兴趣了，感觉提不起兴趣不知道为啥，好像是少了一点神秘感、高深感好像是。。。🥲 想到程序员的高工资，确实是很大的激励，用爱发电无法持久啊，按说棋手下棋虽然下不过 ai，但是可以向 ai 学习进步，棋手按说应该都爱好下棋，应该影响不了心情，其实没那么简单，轮到自己，我感到 10 美元一个月的 ai 能做了，我在这方面就竞争不过 gpu 了，我就拿不到万薪了，我就没那么积极了。这是一条逻辑链上的。
